# PolyCoffee – Claude Code Guide

PolyCoffee is a Java-based Point of Sale (POS) system for a coffee shop chain.
It supports table-side ordering, cashier workflows, admin management,
self-order kiosks, a guest loyalty/voucher system, and statistical reporting.

## Project Structure

- `src/main/java/chez1s/assignment/controller/` – Jakarta Servlet controllers (MVC front controllers)
- `src/main/java/chez1s/assignment/service/` – Business logic layer
- `src/main/java/chez1s/assignment/repository/` – Hibernate ORM data access
- `src/main/java/chez1s/assignment/entity/` – JPA entity classes
- `src/main/java/chez1s/assignment/dto/` – Data Transfer Objects
- `src/main/java/chez1s/assignment/filter/` – Servlet filters (auth, CORS, etc.)
- `src/main/java/chez1s/assignment/util/` – Utilities (MinIO, Gson helpers, etc.)
- `src/main/webapp/views/` – JSP view templates grouped by feature
- `src/main/webapp/static/` – Static assets (CSS, JS)
- `src/main/resources/` – `messages.properties`, Hibernate/META-INF config
- `docker-compose.yml` – MySQL 8 + MinIO object storage services
- `init.sql` – Database seed/schema script

## Build & Run

```bash
# Start infrastructure
docker-compose up -d

# Build WAR
./mvnw clean package

# Deploy to Tomcat 11 (Jakarta EE 10)
# Drop target/Assignment-1.0-SNAPSHOT.war into $TOMCAT_HOME/webapps/
```

## Key Conventions

- Servlet mappings are declared in `src/main/webapp/WEB-INF/web.xml`.
- Session-based authentication; roles: `admin`, `cashier`, `guest`.
- All JSON responses use Gson; REST-style API endpoints are in `*ApiController` servlets.
- Images are stored in MinIO; `ImageServlet` proxies them.
- I18n strings live in `messages.properties` (Vietnamese + English).
- Lombok is used project-wide; ensure annotation processing is enabled in your IDE.
